目标：输出面向芯片架构师 / 互连系统设计团队 / AI Infra 投研团队的技术深度分析报告。

## 核心分析框架：四层分解法

对任何超级芯片系统，按以下四层逐层分解分析：

### Layer 1: 计算层（Compute Die）
- 计算芯片的制程节点、面积、晶体管数
- 计算密度的瓶颈（算力/面积 vs 算力/功耗）
- 与通信层的接口（UCIe / NVLink-C2C / 私有协议）

### Layer 2: 通信/路由层（Interconnect / Routing）
- 片间互连拓扑（3D Torus / Fat-Tree / All-to-All / Ring）
- 交换芯片（Switch ASIC）的角色与规格
- 协议栈（NVLink / UALink / ICI / 以太网 / InfiniBand）
- 是否从计算 die 解耦成独立芯片（这是范式级变化）

### Layer 3: I/O 后端层（I/O Backend）
- SerDes 速率与通道数
- PHY 类型（电 / 光 / CPO）
- PCIe / CXL 控制器
- 内存控制器（HBM / DDR5 / LPDDR）

### Layer 4: 内存子系统层（Memory Subsystem）
- HBM 代数与堆叠层数
- 内存带宽 / 容量 / 功耗
- CXL 内存池化（如果有）
- 近存计算（如果有）

## 关键提醒

- **物理层优先**：任何 "宣称" 的带宽，先算 `SerDes 速率 × 通道数` 验证物理可实现性
- **距离约束**：224G PAM4 铜线有效距离 ~1m，超过必须论证光方案
- **解耦信号**：当看到通信逻辑从计算 die 剥离成独立 ASIC，这是范式级变化，必须深入分析驱动力和影响
- **数字必须带来源**：每个量化数字注明出处（URL + 日期）
- **避免营销语言**：不要复述 "revolutionary" "industry-leading" 等形容词，用数字说话
